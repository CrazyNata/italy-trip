# Нативное Android-приложение

Это отдельный Kotlin-проект на Jetpack Compose, не использующий Expo, React Native или WebView. Он подключается к тому же проекту Supabase и соблюдает его RLS-права.

## Первый запуск

1. Установите Android Studio: https://developer.android.com/studio
2. В мастере установки выберите Android SDK, Android SDK Platform 35 и Android SDK Platform-Tools.
3. Откройте именно каталог `android/` в Android Studio и согласитесь на загрузку Gradle-зависимостей.
4. Подтвердите RSA-ключ отладки на телефоне, затем выберите телефон в списке устройств Android Studio и нажмите Run.

Для регистрации с подтверждением email добавьте в Supabase Auth -> URL Configuration дополнительный Redirect URL: `italytrip://login-callback`.

Android Studio создаст Gradle Wrapper при первом импорте, если предложит это сделать. Минимальная версия Android для приложения - 8.0 (API 26).

## Реализовано

- Вход и регистрация через Supabase Auth.
- Сохранение сессии средствами Supabase Kotlin SDK.
- Загрузка текущей поездки из `trip_state` с идентификатором `main`.
- Проверка роли владельца через `admins` с существующими RLS-политиками.
- Нативные Compose-экраны: обзор, маршрут, жилье, места, рестораны, бюджет и фото.
- Нативная Mapbox-карта маршрута, мест и ресторанов; runtime-токен загружается из `app_config`.
- Геолокация по явному запросу пользователя с системным разрешением Android.
- Общий фотоальбом из жилья, мест, ресторанов и снимков поездки.
- Системный Photo Picker, EXIF-дата/GPS, уменьшение до 1600/400 px и загрузка в Storage.
- Полные редакторы основных записей, локальный кэш, автосохранение и опрос обновлений.
