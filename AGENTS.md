# Правила репозитория

## Приложение

- Это статическое приложение GitHub Pages: `index.html` содержит интерфейс, стили и логику данных поездки; `auth.js` закрывает доступ через Supabase Auth; `sync.js` синхронизирует ключ `italy_trip` из `localStorage` с Supabase.
- Открывайте `index.html` в браузере. Нужен интернет для CDN-зависимостей, Supabase, Mapbox, шрифтов и иконок; локальной сборки, менеджера пакетов и автотестов нет.
- `support.js` сгенерирован. Исходников и сборки для него в репозитории нет, поэтому не редактируйте его вручную.
- Вышедшие из аккаунта пользователи и не-владельцы имеют доступ только для чтения. Не нарушайте контракт `window.__tripReadOnly`, `window.__isOwner`, `window.__authed` и `window.__onAuthed` между `index.html`, `auth.js` и `sync.js`.
- Токен Mapbox получает `auth.js` из строки `app_config` в Supabase после входа, а не из Git. Инициализация карты в `index.html` должна ждать `window.__MAPBOX_TOKEN`.

## Деплой

- Push в `main` разворачивает корень репозитория на GitHub Pages через `.github/workflows/deploy.yml`.
- Workflow заменяет все `__BUILD__` в `index.html` на SHA коммита. Сохраняйте `?v=__BUILD__` у локальных JavaScript-ресурсов, чтобы после деплоя сбрасывался кэш браузера.

## Процесс работы

- `README.md` является источником истины; проект Claude Design устарел, его не нужно синхронизировать.
- Все изменения сразу коммитятся и отправляются напрямую в `main` («мастер»); отдельные feature-ветки не используются.
